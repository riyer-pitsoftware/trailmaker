# Trailmaker                                                                                                                                                                                                
                                                                                                                                                                                                              
  ## What This Is                                                                                                                                                                                             
                                                                                                                                                                                                              
  A reusable system for generating interactive, educational "trails" through open-source codebases. A model scans a project, extracts insights, maps technology dependencies 3 levels deep, and produces a    
  `trail.json`. A generic HTML viewer renders the trail as an Oregon Trail-inspired experience with branch trails, puzzles, and progress tracking.                                                            
                                                                                                                                                                                                              
  ## Origin                                                                                                                                                                                                   
                                                                                                                                                                                                              
  Extracted from [claude-harness](https://github.com/riyer-pitsoftware/claude-harness), which is an interactive trail through the Claude Code CLI source (512K lines of TypeScript). That project remains the 
  first trail instance. This project owns the generic viewer and generator.                                                                                                                                   
                                                                                                                                                                                                              
  ## Architecture                                                                                                                                                                                             
                                                                                                                                                                                                              
  trail.json           ← model-generated, one per project                                                                                                                                                     
  index.html           ← generic viewer (engine + UI), project-agnostic                                                                                                                                       
  generate-trail.md    ← prompt/instructions for the model scan pipeline                                                                                                                                      
  schema/trail.schema.json ← JSON Schema for trail.json validation                                                                                                                                            
                                                                                                                                                                                                              
  ### trail.json — The Data Contract                                                                                                                                                                          
                                                                                                                                                                                                              
  ```json                                                                                                                                                                                                     
  {                                                                                                                                                                                                           
    "meta": {                                                                                                                                                                                                 
      "title": "The <Project> Trail",                                                                                                                                                                     
      "subtitle": "One-line description",                                                                                                                                                                     
      "stats": "Key stats string",                                                                                                                                                                            
      "repo": "https://github.com/org/repo",                                                                                                                                                                  
      "icon": "emoji"                                                                                                                                                                                         
    },                                                                                                                                                                                                        
    "chapters": [                                                                                                                                                                                             
      {"id": "slug", "title": "Chapter Title", "desc": "What you'll find here"}                                                                                                                               
    ],                                                                                                                                                                                                        
    "insights": [                                                                                                                                                                                             
      {                                                                                                                                                                                                       
        "ch": 0,                                                                                                                                                                                              
        "t": "Insight Title",                                                                                                                                                                                 
        "d": "What was found — specific, with real numbers",                                                                                                                                                  
        "f": "path/to/source/file.ts",                                                                                                                                                                        
        "l": "<strong>Why it matters:</strong> Educational explanation",                                                                                                                                      
        "v": "optional value badge text"                                                                                                                                                                      
      }                                                                                                                                                                                                       
    ],                                                                                                                                                                                                        
    "branches": {                                                                                                                                                                                             
      "0": {                                                                                                                                                                                                  
        "trail": [                                                                                                                                                                                            
          {"title": "Sourced Tech Name", "level": "Sourced Tech", "desc": "...", "core": "..."},                                                                                                              
          {"title": "How It Works Name", "level": "How It Works", "desc": "...", "core": "..."},                                                                                                              
          {"title": "Core Tech Name", "level": "Core Tech", "desc": "...", "core": "..."}                                                                                                                     
        ],                                                                                                                                                                                                    
        "puzzle": {"q": "Question text", "opts": ["A", "B", "C", "D"], "ans": 0}                                                                                                                              
      }                                                                                                                                                                                                       
    }                                                                                                                                                                                                     
  }
                                                                                                                                                                                                              
  - insights[].ch is the 0-based index into the chapters array                                                                                                                                                
  - branches keys are stringified insight indices                                                                                                                                                             
  - Branch trails are always exactly 3 levels deep: sourced tech → how it works → core CS/math concept                                                                                                        
  - Puzzles are always 4-option MCQ with ans as 0-based index of the correct option                                                                                                                           
                                                                                                                                                                                                              
  Viewer (index.html)                                                                                                                                                                                         
                                                                                                                                                                                                              
  The viewer is a single self-contained HTML file. No build step, no dependencies. Hosts on GitHub Pages as-is.                                                                                               
                                                                                                                                                                                                              
  Loading: reads trail.json via fetch(). Supports a ?trail=path/to/trail.json query param. Default: trail.json in same directory.                                                                             
                                                                                                                                                                                                              
  Features:                                                                                                                                                                                                   
  - Trail map with dots, segments, chapter markers, and wagon position indicator                                                                                                                              
  - Card-based insight display with file source links                                                                                                                                                         
  - Branch trail overlay (3-level tech deep-dive) with mini trail map                                                                                                                                         
  - Puzzle gate to return from branch to main trail                                                                                                                                                           
  - Chapter interstitials, search overlay, chapter nav sidebar                                                                                                                                                
  - localStorage progress persistence (namespaced per project via meta.id or meta.repo)                                                                                                                       
  - Keyboard navigation (arrows, vim keys, / search, C chapters, G jump, [ ] chapter skip)                                                                                                                    
  - Touch swipe support                                                                                                                                                                                       
  - CRT scanline visual theme                                                                                                                                                                                 
                                                                                                                                                                                                              
  File source links: meta.repo is used to construct GitHub links. When location.hostname includes github.io, links point to repo/blob/main/path (files) or repo/tree/main/path (directories). Otherwise links 
  are relative paths for local use.                                                                                                                                                                           
                                                                                                                                                                                                              
  Generator (generate-trail.md)                                                                                                                                                                               
                                                                                                                                                                                                              
  A system prompt / instruction set for a model to scan a repo and produce trail.json. Four phases:                                                                                                           
                                                                                                                                                                                                              
  1. Recon — tree, file counts, language stats, README, manifest files → project metadata + candidate chapter boundaries                                                                                      
  2. Chapter mapping — Group codebase into 8-20 logical chapters (subsystems, not just directories)                                                                                                           
  3. Insight extraction — Per chapter: magic numbers, surprising patterns, security boundaries, performance tricks, unusual dependencies, clever algorithms. 5-15 per chapter. Each must reference a real file
   path.                                                                                                                                                                                                      
  4. Branch generation — For key insights, trace the tech chain 3 deep. Write one puzzle per branch.                                                                                                          
                                                                                                                                                                                                              
  Design Decisions                                                                                                                                                                                            
                                                                                                                                                                                                              
  ┌────────────────────────┬───────────────────────┬──────────────────────────────────────────────────────────────────────────┐                                                                               
  │        Decision        │        Choice         │                                Rationale                                 │                                                                           
  ├────────────────────────┼───────────────────────┼──────────────────────────────────────────────────────────────────────────┤
  │ Data format            │ JSON, not embedded JS │ Parseable, validatable, diffable, model-friendly                         │
  ├────────────────────────┼───────────────────────┼──────────────────────────────────────────────────────────────────────────┤                                                                               
  │ One file per project   │ trail.json            │ Simple versioning, no database                                           │                                                                               
  ├────────────────────────┼───────────────────────┼──────────────────────────────────────────────────────────────────────────┤                                                                               
  │ No build step          │ Raw HTML + fetch      │ Hosts on any static server, including GH Pages                           │                                                                               
  ├────────────────────────┼───────────────────────┼──────────────────────────────────────────────────────────────────────────┤                                                                               
  │ Branch depth           │ Always 3              │ Deep enough for fundamentals, shallow enough to stay focused             │                                                                               
  ├────────────────────────┼───────────────────────┼──────────────────────────────────────────────────────────────────────────┤                                                                               
  │ Puzzle format          │ 4-option MCQ          │ Easy to generate, unambiguous right/wrong                                │                                                                               
  ├────────────────────────┼───────────────────────┼──────────────────────────────────────────────────────────────────────────┤                                                                               
  │ localStorage namespace │ Per meta.repo hash    │ Multiple trails on same domain don't collide                             │                                                                               
  ├────────────────────────┼───────────────────────┼──────────────────────────────────────────────────────────────────────────┤                                                                               
  │ Theme                  │ CRT/retro terminal    │ Distinctive, works in dark mode, pixel fonts are readable at small sizes │                                                                               
  └────────────────────────┴───────────────────────┴──────────────────────────────────────────────────────────────────────────┘                                                                               
                                                                                                                                                                                                              
  Relationship to claude-harness                                                                                                                                                                              
                                                                                                                                                                                                              
  claude-harness is Trail Instance #1. Once this project is ready:                                                                                                                                            
  1. claude-harness/index.html will be replaced by a copy (or iframe/redirect) of this viewer                                                                                                                 
  2. claude-harness/trail.json will contain the Claude Code trail data (currently hardcoded in its index.html)                                                                                                
  3. This project will not contain any Claude Code-specific content                                                                                                                                           
                                                                                                                                                                                                              
  Tech Stack                                                                                                                                                                                                  
                                                                                                                                                                                                              
  - Pure HTML/CSS/JS (no framework, no bundler)                                                                                                                                                               
  - Google Fonts: Press Start 2P (pixel headings), VT323 (terminal body text)                                                                                                                                 
  - localStorage for persistence                                                                                                                                                                              
  - JSON Schema for trail.json validation                                                                                                                                                                     
                                                                                                                                                                                                              
  Development                                                                                                                                                                                                 
                                                                                                                                                                                                          
  # Serve locally                                                                                                                                                                                             
  python3 -m http.server 8000                                                                                                                                                                                 
  # or                                                                                                                                                                                                        
  npx serve .                                                                                                                                                                                                 
                                                                                                                                                                                                              
  # Validate a trail.json                                                                                                                                                                                     
  node validate.js path/to/trail.json                                                                                                                                                                         
                                                                                                                                                                                                              
  # Generate a trail for a new project                                                                                                                                                                        
  claude --system-prompt generate-trail.md "Scan github.com/org/repo"
